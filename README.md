# DevNet_Marathon_NornirMAC

DevNet Marathon Homework. Day 2

Задача:  
Найти заданный MAC адрес в данной топологии. Решение должно работать при изменении топологии и/или количества устройств в сети при сохранении ролей интерфейсов (trunk между коммутаторами, access для клиентских устройств).
Входные данные для задачи:  
MAC-адрес клиента (R2-R3), шлюза по умолчанию (R1) или SVI (SW1-SW4)  
Выходные данные:  
Имя коммутатора и исходящий интерфейс, к которому подключен к клиент с заданным MAC-адресом.  
Для решения задачи необходимо использовать framework Nornir.  

--------------------------------------------------------------------------------------------

Реализация:

Скрипт:
- NornirMAC.py

Входные данные:
- в yaml-файле config.yaml указываются все MAC адреса, подлежащие поиску в сетевой топологии
- в файлах hosts.yaml и groups.yaml в папке .\inventory задается топология (данные для подключения к устройствам)

Выходные данные:
- в консоль выводится отчет о найденных MAC адресах, их связи с устройством и интерфейсом (см. условия Задачи) 

Особенности:
- используется функционал Nornir
- дополнительно разбирается ситуация c voice vlan интерфейсами (исключаются дубли) 

Пример:  

MAC d4ad.71ee.9a46 belongs to device 'test-sw' SVI Vlan405  
MAC 0cd0.f860.eb43 belongs to device 'T9D-ACC1' SVI Vlan443  
MAC 00aa.6e0e.0dec was learned on device 'test-sw' access port Gi0/3  
MAC 8030.e030.23b4 was learned on device 'T9D-CSW1' access port Gi1/0/2  
Press any key to continue . . .  

P.S. В файле output_comments.txt приводятся данные с устройств и комментарии по всем MAC-адресам из списка input_macs.yaml
